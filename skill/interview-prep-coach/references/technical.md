# Technical Interview Reference

## Scorecard Areas

For technical roles, inspect the job description for:

- Core language, framework, or platform depth.
- System design or architecture expectations.
- Debugging and production ownership.
- Data structures, algorithms, or coding fluency.
- Security, reliability, scalability, and observability.
- Collaboration with product, design, data, operations, or customers.

## Practice Types

Use the interview stage to pick drills:

- Coding screen: problem clarification, complexity, edge cases, tests.
- Technical deep dive: project architecture, tradeoffs, incident learning.
- System design: requirements, APIs, data model, scale, failure modes.
- Debugging: symptoms, hypotheses, instrumentation, isolation, fix, prevention.
- Senior/staff loop: ambiguity, technical strategy, mentoring, influence.

## Technical Deep Dive Template

Ask for or reconstruct:

- Problem and constraints.
- Architecture or design.
- Key tradeoffs.
- Alternatives rejected.
- Failure modes.
- Testing and rollout.
- Measured impact.
- What the user would change now.

## Coding Interview Coaching

Coach the user to:

- Restate the problem.
- Ask clarifying questions.
- Start with a simple correct approach.
- Name complexity.
- Cover edge cases.
- Talk through tests.
- Improve only after correctness is clear.

Do not provide a memorized solution when mock interviewing unless the user asks
for coaching mode.
